# Google + AI SEO Operating System

This file is the working system for improving Wholelychit website visibility in Google Search, Google video results, YouTube, and AI-powered answer surfaces.

## Honest Rule

No one can guarantee a permanent top Google ranking.

The goal is to build pages and videos that are more useful, clearer, safer, faster, and better connected than competing pages.

Do not use tricks that depend on spam, fake engagement, mass AI pages, cloaking, keyword stuffing, or copied content.

## What To Ignore

Avoid any tactic that promises guaranteed rankings through:

- auto-generated bulk pages
- fake backlinks
- fake traffic or click bots
- auto-click browser scripts
- keyword stuffing
- hidden text
- copied YouTube transcripts without added value
- AI articles published without human review
- doorway pages across many domains

These can create ranking risk and trust risk.

## Core Ranking System

For each website, build around three layers:

1. Main hub page
2. Supporting answer pages
3. Short videos embedded on the best pages

Each page must answer one real question and link naturally to the next useful step.

## Page Standard

Every important page should include:

- one clear H1
- one direct answer near the top
- short paragraphs
- useful headings
- internal links to related pages
- a clear next step
- plain-language trust/safety notes when needed
- unique title and meta description
- canonical URL
- mobile-friendly layout
- fast loading

## Video SEO Standard

For pages with videos:

- embed the video visibly on the page
- add a short written summary below the video
- add a transcript or lesson notes when useful
- use a stable thumbnail
- make sure the page itself is indexable
- add VideoObject structured data when possible
- include the video page in sitemap/video sitemap when the site supports it

YouTube videos should have:

- title that matches one real search query
- first two description lines linking to the matching website page
- plain-language description
- chapters when useful
- pinned comment linking to the website page
- accurate captions or transcript
- thumbnail that says the exact problem solved

## AI Search Standard

To be easier for AI answers and search summaries to understand, pages should include:

- a concise answer block
- numbered steps when the user needs a process
- FAQ sections for repeated questions
- examples, checklists, or tables where helpful
- clear author/site purpose and trust language
- updated dates only when the content is actually reviewed

AI can draft pages, but every page needs human-style editing for usefulness, safety, and accuracy.

## Weekly Workflow

Every week:

1. Choose one project.
2. Choose one hub or supporting topic.
3. Create or improve one page.
4. Create one short video script for that page.
5. Add internal links from 2-4 related pages.
6. Check title, meta description, sitemap, and mobile readability.
7. Record the change in `PROJECT-STATUS.md`.

## Monthly SEO Review

Every month:

- check Google Search Console for pages with impressions but low clicks
- rewrite weak titles/descriptions
- add FAQ sections to pages getting impressions
- add internal links to pages that should rank
- prune or improve thin pages
- update stale facts
- create one video for the page with the best potential

## AI Use Rules

Use AI for:

- outlines
- FAQ ideas
- title variations
- summaries
- scripts
- internal-link suggestions
- schema drafts

Do not use AI for:

- mass publishing without review
- fake expertise
- copied competitor rewrites
- legal, financial, medical, gambling, or trading claims without caution
- fake testimonials or fake experience
