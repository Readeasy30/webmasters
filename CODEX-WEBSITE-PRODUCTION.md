# Codex Website Production System

Use Codex as the production helper for Wholelychit websites.

## Goal

Build useful websites faster without breaking working code.

## Core workflow

1. Pick one website repository.
2. Read the repo instructions first.
3. Check the current files.
4. Make safe improvements.
5. Commit changes.
6. Report what changed.
7. Continue to the next safe improvement.

## Repository order

1. `readeasy30.com`
2. `matheasy30.com`
3. `my-petneeds`
4. `restaurantaibot.com`
5. `slotsfreeusa.com`
6. `marketing-system`

## Codex should do

- fix broken links
- improve page copy
- improve mobile layout
- add missing support pages
- add simple checklists
- improve README files
- improve AI instruction files
- add safe HTML sections
- improve CSS carefully
- improve JavaScript without changing the stack
- keep each site simple and fast

## Codex should not do

- switch to React
- switch to Vite
- add build tools without approval
- delete major working code
- add private keys
- add payment setup
- add live ad code
- add live analytics code without approval
- mix website code between repos

## Standard site files

Each public website should have:

- `README.md`
- `AGENT-INSTRUCTIONS.md`
- `index.html`
- `css/style.css`
- about page
- contact page when needed
- privacy page when needed
- terms page when needed
- disclosure page when monetized

## Standard build steps

For each site:

1. Make homepage clear.
2. Make navigation simple.
3. Make mobile layout strong.
4. Add plain-language copy.
5. Add trust pages.
6. Add topic pages.
7. Add monetization plan later.
8. Test basic links and buttons.

## Time rule

Do not stop after one small edit when safe next work is clear.

Work in groups of safe commits, then report:

- files changed
- what improved
- blocked items
- next work

## Safety rule

If a file write is blocked, simplify the wording and try again.

If a risky change is requested, keep the working version and make a safer smaller update.
