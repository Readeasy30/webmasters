# Agent Instructions

## Primary job

This repository is the central marketing, SEO, content, launch, monetization-planning, and review-control system for Wholelychit projects.

Do not build a public website here unless a public website already exists in this repo or Gerry clearly requests one later.

## Main rule

AI and Codex must do normal safe file work directly.

Create, update, replace, repair, and commit safe files yourself when write access is available.

Do not ask Gerry to paste files, create files manually, replace files manually, or update files manually.

Ask Gerry only for account-level actions AI cannot perform, such as sign-ins, billing approval, domain ownership, Cloudflare permissions, connector repair, or third-party service approval.

## Continuous safe-work rule

Do not stop after one safe change.

Continue through the safe queue until blocked by one of these:

1. Missing write access.
2. Missing required account permission.
3. Private keys, tokens, payment setup, live ads, or live tracking would be required.
4. A framework change or major code deletion would be required.
5. A production-breaking change cannot be safely repaired in the same session.

Commit each useful safe change.

Report only after several useful commits or a real blocker.

## Repository boundary

This repo owns planning and repeatable systems.

Website repos own production site code.

Connected education repos:

- `Wholelychit/readeasy30.com`
- `Wholelychit/matheasy30.com`
- `Wholelychit/marketing-system`

Use this repository to coordinate shared education marketing, campaigns, SEO, social copy, launch plans, and status notes.

## Safe work allowed here

- Markdown planning files
- SEO plans
- content calendars
- campaign files
- social post templates
- YouTube Shorts scripts
- Canva asset plans
- launch checklists
- monetization plans before implementation
- Codex instructions
- review/fix loop files
- blocker logs

## Do not add here

- private keys
- API tokens
- payment setup
- live tracking scripts
- live ad scripts
- unrelated website code
- framework migrations
- unsupported legal, medical, financial, safety, or earnings guarantees

## Required review process

For each active website repo:

1. Read README.md.
2. Read AGENTS.md.
3. Read AGENT-INSTRUCTIONS.md if present.
4. Read LOCKED-CHECKPOINT.md if present.
5. Read FILE-MANAGEMENT.md if present.
6. Read PROJECT-STATUS.md if present.
7. Check homepage title, meta description, canonical URL, navigation, footer, legal links, robots.txt, sitemap.xml, and mobile usability.
8. Fix safe issues directly.
9. Commit useful changes.
10. Record blockers instead of stopping the queue.
11. Continue until the safe queue is complete or actually blocked.

## Reporting rule

Report files changed, useful commits made, improvements, real blockers, and next safe queue.

Do not ask Gerry to continue after one safe update.
